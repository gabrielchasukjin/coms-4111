Performance & Systems Design

Q1 According to Amdahl's Law, if 80% of a program's runtime can be parallelized (p = 0.8) and you improve that portion with speedup S → ∞, what is the absolute upper bound on overall speedup?
A. 5× (upper bound is 1 / (1 − p))
B. 10× (upper bound is 1 / p)
C. Unlimited
D. 1.8× (average of the parallelizable and sequential fractions)
Correct Answer
A. 5× (upper bound is 1 / (1 − p))
Amdahl's Law: Speedup = 1 / ((1 − p) + p/S). As S → ∞, the p/S term vanishes, leaving 1 / (1 − 0.8) = 1 / 0.2 = 5. The sequential portion (20%) is the bottleneck — no matter how fast you make the parallel part, you can never exceed 5×.
Q2 The course describes throughput and latency as orthogonal metrics. Which example best illustrates that optimizing one does not automatically improve the other?
A. A faster CPU that reduces both the time per request and the number of requests processed per second simultaneously
B. A pizzeria that adds more ovens (raises throughput) but still makes each pizza take just as long to bake (latency unchanged)
C. A network switch that increases bandwidth and reduces round-trip time in equal proportion
D. A disk that reads data faster while also reading more bytes per second
Correct Answer
B. A pizzeria that adds more ovens (raises throughput) but each pizza still takes just as long to bake (latency unchanged)
Throughput = how many items per unit time. Latency = how long one item takes. Adding ovens increases pizzas/hour (throughput) but each pizza still bakes for the same duration (latency). The other options conflate the two or improve both simultaneously.
Q3 When Petflix fans out one user request to four microservices (Auth, Playback, Recommendations, Analytics) simultaneously via gRPC, the total latency experienced by the user equals:
A. The sum of all four service response times
B. The average of the four service response times
C. The slowest of the four individual service response times
D. The fastest of the four individual service response times
Correct Answer
C. The slowest of the four individual service response times
When requests fan out in parallel, you must wait for all to complete. The total latency = max(all response times). You're only as fast as your slowest service.
Q4 In the Petflix case study, keeping DNS TTL values very short (e.g., 30 seconds) trades off which two properties against each other?
A. Security and availability
B. Compression efficiency and bandwidth
C. Encryption strength and query speed
D. Failover speed versus DNS query volume
Correct Answer
D. Failover speed versus DNS query volume
Short TTL = clients re-resolve DNS frequently, so if a server goes down, traffic reroutes quickly (fast failover). But this means more DNS queries hitting the infrastructure. Long TTL = fewer queries but slower failover.

SQL Basics

Q5 In the SQL SELECT-FROM-WHERE model, the term "selection" refers to:
A. Choosing which columns (attributes) appear in the output table
B. Specifying which tables are involved in the query using the FROM clause
C. Filtering a relation's rows based on a condition in the WHERE clause, keeping only rows that satisfy the predicate
D. Sorting and deduplicating the result set before returning it to the client
Correct Answer
C. Filtering a relation's rows based on a condition in the WHERE clause
In relational algebra, selection (σ) = filtering rows. Projection (π) = choosing columns. Common trap: the SQL keyword SELECT actually does projection, while the WHERE clause does selection.
Q6 When ORDER BY Price, PName is used and two rows have the same Price, the ordering between those tied rows is determined by:
A. The value of PName as the secondary sort key
B. The physical insertion order of the rows in the table
C. Random assignment
D. The primary key of the table, applied automatically by the database engine
Correct Answer
A. The value of PName as the secondary sort key
ORDER BY Price, PName sorts first by Price, then breaks ties using PName. The second column listed is the tiebreaker.
Q7 Which of the following correctly describes SQL's case sensitivity rules as covered in the course?
A. Both SQL keywords and string literal values are fully case-insensitive
B. SQL keywords are case-sensitive, but string literal values are case-insensitive
C. SQL keywords (SELECT, FROM, WHERE) are case-insensitive, but string values are case-sensitive
D. Case sensitivity is identical for both keywords and values, both requiring exact case matching
Correct Answer
C. SQL keywords are case-insensitive, but string values are case-sensitive
SELECT, select, and SeLeCt are all the same. But 'Smith' ≠ 'smith' in string comparisons. Keywords are structural; string literals are data.

Serialization & Schema Design

Q8 Protocol Buffers produce a smaller on-wire payload than JSON primarily because they:
A. Use integer field tags instead of repeated string field names, and encode integers as varints rather than ASCII digit sequences
B. Automatically apply gzip compression during serialization before any bytes are sent
C. Strip all optional fields from the payload, transmitting only mandatory fields
D. Limit field values to fixed 32-bit or 64-bit binary representations, regardless of the actual value magnitude
Correct Answer
A. Integer field tags + varint encoding
JSON repeats field names as strings in every object ("name": "..."). Protobuf replaces names with small integer tags and uses varints — variable-length encoding where small numbers take fewer bytes. No compression is applied automatically.
Q9 In the Uber Docstore design, the phrase "flexible schema, not no schema" means:
A. Schema definitions are written per-row at insertion time, with no global table-level schema enforced anywhere
B. Consistency constraints and schema logic are centralized at the system layer, allowing the data format to evolve while still maintaining coherent cross-row guarantees
C. The database engine auto-detects schema from observed query patterns and adjusts column types at runtime
D. Tables may have an unlimited number of columns, with each column's type determined at query time
Correct Answer
B. Consistency constraints centralized at the system layer, allowing evolution while maintaining guarantees
"Flexible schema" doesn't mean no rules at all. It means the schema can evolve (add fields, change types) without downtime, but constraints and validation are still enforced at a centralized layer.
Q10 FOREIGN KEY (student_id) REFERENCES Students(cuid) is declared on the Enrolled table. An INSERT into Enrolled specifies a student_id that does not exist in Students. What happens?
A. The row is inserted with student_id silently set to NULL
B. The row is inserted but a warning is logged in the error log
C. A new placeholder row is automatically created in Students to satisfy the constraint
D. The INSERT is rejected
Correct Answer
D. The INSERT is rejected
Foreign keys enforce referential integrity: every value in the child column must exist in the parent column. If the referenced value doesn't exist, the database refuses the insert outright.
Q11 Which of the following is NOT a valid DBA-configurable option for handling deletion of a parent row referenced by a child table's foreign key?
A. Cascade the deletion to all referencing child rows (ON DELETE CASCADE)
B. Set the child rows' foreign key column to NULL (ON DELETE SET NULL)
C. Automatically re-create the deleted parent row to preserve referential integrity
D. Disallow the deletion entirely if any child rows reference the parent (ON DELETE RESTRICT)
Correct Answer
C. Automatically re-create the deleted parent row (not a real option)
Valid options: CASCADE, SET NULL, SET DEFAULT, RESTRICT / NO ACTION. There is no "auto-recreate parent" option — that is not part of the SQL standard.

Joins & Aggregation

Q12 According to the course, the cross-product model of SQL join semantics (compute all combinations, then filter on the WHERE condition) describes:
A. The actual physical execution strategy used internally by all relational databases
B. An approximation that applies only to OUTER JOINs, not INNER JOINs
C. The logical definition of what a join means — real databases use far more efficient internal strategies such as hash joins or index nested-loop joins
D. The execution model for subqueries in the FROM clause only
Correct Answer
C. The logical definition — real databases use far more efficient strategies
The cross-product model is a conceptual / logical definition: "imagine all combinations, then filter." Real databases never actually materialize the full cross product — they use hash joins, merge joins, index lookups, etc.
Q13 The maximum number of rows that can result from an INNER JOIN between a table with L rows and a table with R rows is:
A. L + R
B. max(L, R)
C. min(L, R)
D. L × R
Correct Answer
D. L × R
In the worst case (every row matches every other row), you get the full Cartesian product: L × R rows.
Q14 The HAVING clause differs from WHERE in that:
A. HAVING filters individual rows before they are grouped; WHERE filters aggregated groups after computation
B. HAVING filters groups after aggregation and can reference aggregate functions (e.g., HAVING COUNT(*) > 5); WHERE filters individual rows before grouping and cannot reference aggregate functions
C. HAVING and WHERE are interchangeable and always produce identical results
D. HAVING can only appear in queries that also use ORDER BY
Correct Answer
B. HAVING filters groups after aggregation; WHERE filters rows before grouping
Execution order: FROM → WHERE (filter rows) → GROUP BY → HAVING (filter groups) → SELECT → ORDER BY. Only HAVING can use aggregate functions like COUNT(*), SUM(), etc.
Q15 The SQL condition price > ALL (SELECT price FROM Product WHERE maker = 'GWorks') is satisfied when:
A. The price exceeds every single value in the GWorks list
B. The price exceeds the average of all GWorks prices
C. The price exceeds at least one GWorks price
D. The subquery returns no rows (vacuously true regardless of price value)
Correct Answer
A. The price exceeds every single value in the GWorks list
> ALL means "greater than every value in the set" — i.e., greater than the maximum. Compare with > ANY / > SOME which means "greater than at least one."
Q16 The SQL EXISTS (subquery) predicate evaluates to TRUE when:
A. The subquery returns at least one row whose values are all non-NULL
B. The subquery returns exactly one row
C. Every row in the outer table has a matching row returned by the subquery
D. The subquery produces at least one row, regardless of the content of that row
Correct Answer
D. The subquery produces at least one row, regardless of the content
EXISTS only checks whether any rows are returned — it doesn't care what's in them. Even a row of all NULLs counts. That's why SELECT 1 or SELECT * inside EXISTS are functionally identical.

Isolation Levels & Anomalies

Q17 A dirty read occurs when:
A. Transaction Tx reads data written by Transaction Ty that has not yet committed
B. Two concurrent transactions write to the same row, causing one update to be permanently lost
C. Transaction Tx reads the same single row twice and gets two different values because Ty committed between reads
D. Transaction Tx re-reads a set of rows and finds new rows inserted by a committed Ty
Correct Answer
A. Tx reads data written by Ty that has not yet committed
Dirty read = reading uncommitted data. If Ty later aborts, Tx has read data that never officially existed. B = lost update, C = unrepeatable read, D = phantom read.
Q18 The "Read Committed" isolation level prevents which anomaly, while still permitting others?
A. It prevents all anomalies including phantom reads, unrepeatable reads, and lost updates
B. It prevents dirty reads only
C. It prevents dirty reads and unrepeatable reads but still allows phantom reads and lost updates
D. It prevents phantom reads only
Correct Answer
B. It prevents dirty reads only
Isolation levels from weakest to strongest: Read Uncommitted (allows everything) → Read Committed (prevents dirty reads) → Repeatable Read (also prevents unrepeatable reads) → Serializable (prevents all anomalies including phantoms).
Q19 A phantom read anomaly occurs when:
A. Transaction Tx reads an uncommitted update written by Transaction Ty
B. Two transactions both write to the same row simultaneously, causing one write to be silently discarded
C. Transaction Tx re-executes the same range query and finds different rows because Ty inserted or deleted rows matching the predicate between the two reads
D. Transaction Tx reads the same individual row twice and gets different values due to Ty updating that specific row
Correct Answer
C. Tx re-executes a range query and finds different rows due to Ty's inserts/deletes
Phantom read = new "ghost" rows appear (or disappear) in a repeated range query. Unlike unrepeatable reads (same row, different value), phantoms involve the set of rows changing. A = dirty read, D = unrepeatable read.

Concurrency Control & Locking

Q20 A conflict graph (precedence graph) is used to test conflict serializability. A schedule is conflict serializable if and only if:
A. The conflict graph contains no self-loops
B. Every edge points from an earlier to a later transaction in submission order
C. The conflict graph has exactly one strongly connected component
D. The conflict graph is acyclic
Correct Answer
D. The conflict graph is acyclic
If there's a cycle (T1 must come before T2, but T2 must come before T1), no valid serial ordering exists. Acyclic = DAG = conflict serializable. A topological sort of the DAG gives the equivalent serial order.
Q21 Strict serializability provides a stronger guarantee than ordinary conflict serializability because it additionally requires that:
A. If transaction Y starts after transaction X commits (they are non-concurrent), then X must precede Y in any equivalent serial order, preserving real-time ordering
B. All transactions must acquire only exclusive locks throughout their entire execution
C. No two transactions may execute concurrently under any circumstances
D. Transactions must commit in strictly increasing order of their start timestamps
Correct Answer
A. If Y starts after X commits, X must precede Y in the equivalent serial order (real-time ordering)
Regular conflict serializability only guarantees equivalence to some serial order. Strict serializability adds: the serial order must respect real-time precedence. If X finished before Y started, X must appear first.
Q22 Under Strict 2PL, when Transaction Tx holds a shared (S) lock on an object, which is true?
A. No other transaction can acquire any lock (S or X) on that object
B. Other transactions may also acquire S locks, but no other transaction may acquire an X (exclusive) lock
C. Tx is automatically permitted to upgrade its S lock to an X lock at any time without waiting
D. The S lock is released as soon as Tx finishes reading the object for the first time
Correct Answer
B. Other transactions may acquire S locks, but not X locks
Shared locks are compatible with other shared locks (multiple readers allowed) but incompatible with exclusive locks (no writer while someone is reading). Lock compatibility: S+S = OK, S+X = BLOCK, X+X = BLOCK.
Q23 In the "Wait-Die" deadlock prevention scheme, what happens when a lower-priority transaction Tx requests a lock currently held by higher-priority transaction Ty?
A. Ty is immediately aborted so that Tx can proceed
B. Both Tx and Ty are aborted and restarted with freshly assigned priorities
C. Tx is aborted (dies)
D. Tx waits for Ty to finish and release the lock naturally
Correct Answer
C. Tx is aborted (dies)
Wait-Die: Older (higher priority) transactions wait; younger (lower priority) transactions die. Here Tx is lower-priority (younger), so it dies. Mnemonic: "Old waits, young dies."
Q24 The waits-for graph used in deadlock detection is correctly defined as:
A. A graph identical to the conflict graph, with edges between any two transactions that access the same data item
B. A graph with a node for each data object and edges from each lock holder to each waiting requester
C. A bipartite graph with transactions on one side and data objects on the other
D. A directed graph with a node per transaction and an edge from Ti → Tj if Ti is waiting for a lock currently held by Tj
Correct Answer
D. Directed graph: node per transaction, edge Ti → Tj if Ti waits for Tj's lock
The waits-for graph has transactions as nodes. An edge Ti → Tj means "Ti is blocked, waiting for Tj to release a lock." A cycle in this graph = deadlock.
Q25 Conservative 2PL (C2PL) prevents deadlocks by requiring that:
A. A transaction must acquire all locks it will ever need before any operations begin. If it cannot obtain all at once, it releases them all and retries later
B. Locks are acquired in a globally defined total ordering of data objects
C. Each transaction is assigned a priority at start time and always preempts lower-priority lock holders
D. Locks are released immediately after each operation rather than waiting until commit
Correct Answer
A. Acquire all locks before any operations begin; release all and retry if unable
Conservative 2PL = "all or nothing" lock acquisition. A transaction declares all locks upfront. If any lock is unavailable, it doesn't hold partial locks — it backs off entirely. No transaction ever holds some locks while waiting for others, so deadlocks are impossible.
Q26 In the "Wound-Wait" deadlock prevention scheme, if higher-priority transaction Tx requests a lock currently held by lower-priority transaction Ty:
A. Tx waits for Ty to finish, since Ty was first to acquire the lock
B. Ty is aborted ("wounded") so that Tx can immediately acquire the lock
C. Both transactions are aborted and restarted
D. Tx is aborted and restarted while Ty continues
Correct Answer
B. Ty is aborted ("wounded") so that Tx can immediately acquire the lock
Wound-Wait: Older (higher priority) transactions wound (abort) younger ones; younger transactions wait for older ones. Here Tx is higher-priority, so it wounds Ty. Mnemonic: "Old wounds, young waits."

Indexing

Q27 The key difference between a dense index and a sparse index is:
A. A dense index is stored in memory; a sparse index is stored on disk
B. A dense index contains an index entry for every search-key value in the file; a sparse index has entries for only some values (e.g., one per block)
C. A dense index supports only equality lookups; a sparse index additionally supports range queries
D. A dense index always corresponds to the primary key; a sparse index always corresponds to a secondary attribute
Correct Answer
B. Dense = entry for every value; Sparse = entries for only some values
Dense: every search-key value has a pointer in the index. Sparse: only some values (typically one per data block) have pointers. Sparse is smaller but requires sequential scan within the block.
Q28 A primary index is defined as:
A. An index whose search key specifies the sequential (physical) ordering of records in the file on disk
B. Any index built on the column declared as PRIMARY KEY in the table schema
C. The index with the highest query hit rate in the workload
D. Any index that enforces uniqueness on its search-key values
Correct Answer
A. An index whose search key specifies the physical ordering of records on disk
A primary (clustering) index matches the physical sort order of the data file. Note: "primary index" ≠ "index on the PRIMARY KEY" — they are different concepts!
Q29 Why must a secondary index always be dense?
A. The SQL standard mandates that all non-primary indexes must have complete key coverage
B. Secondary indexes are always stored in memory
C. Secondary indexes are always built on non-unique attributes
D. Because the secondary index's search key does not determine the physical file ordering
Correct Answer
D. The search key does not determine the physical file ordering
A sparse index works by finding the nearest entry and then scanning sequentially. But if the file isn't sorted by that key, sequential scan won't find the right record — it could be anywhere on disk. So secondary indexes must be dense.
Q30 To look up a record with search-key value K using a sparse index, the procedure is:
A. Perform a binary search directly on the raw data file, bypassing the index
B. Scan the entire index sequentially from beginning to end, then fetch the matching record
C. Find the index entry with the largest search-key value ≤ K, then scan the data file sequentially from that point
D. Hash K to find its block number and read that block directly from disk
Correct Answer
C. Find largest key ≤ K in the index, then scan data sequentially from there
Since a sparse index only has some keys, you find the floor entry (largest key ≤ K). That entry points to the right block, then you scan forward in the sorted data file until you find K.
Q31 The primary motivation for building a multilevel index is:
A. To enable range query support that a single-level flat index cannot provide
B. To enforce referential integrity constraints across multiple tables
C. To reduce the number of distinct search key values maintained as the table grows
D. To handle the case where the index itself is too large to fit entirely in memory
Correct Answer
D. The index itself is too large to fit in memory
If your index has millions of entries, it doesn't fit in RAM. Solution: build an index on the index. The upper-level index is smaller and can fit in memory. This is exactly the idea behind B-trees.

Bloom Filters

Q32 A Bloom filter can produce false positives but never false negatives. In practical terms, this means:
A. "Definitely not in the set" is always correct (skip disk); "might be in the set" needs confirmation via disk lookup
B. "In the set" is guaranteed to be present — no further verification needed
C. False positives and false negatives occur at the same rate
D. The filter perfectly detects absent items but occasionally misses present items
Correct Answer
A. "Definitely not" is always correct; "might be" needs confirmation
Bloom filters are great as a negative filter: "No" means definitely no (skip the expensive lookup). "Yes" means maybe yes (still check). This saves work by eliminating guaranteed misses before they hit disk.
Q33 A Bloom filter uses k hash functions. When checking membership of element x, the filter returns "possibly present" if and only if:
A. At least one of the k hash positions is set to 1
B. All k hash positions are set to 1
C. More than half of the k hash positions are set to 1
D. The bitwise XOR of all k hash positions equals a predefined checksum value
Correct Answer
B. All k hash positions must be set to 1
On insert: set all k bit positions to 1. On lookup: check all k positions. If all are 1, the item is possibly present. If any is 0, the item is definitely absent.
Q34 Why do standard Bloom filters not support deletion of elements?
A. Multiple elements may hash to the same bit positions, so clearing a bit could cause other elements to be falsely reported as absent
B. Deletion requires re-hashing all remaining elements, which is computationally prohibitive
C. The hash functions are one-way, so given a set bit it's impossible to determine which elements caused it
D. Deletion would require tracking exact insertion counts, which the standard structure doesn't maintain
Correct Answer
A. Multiple elements share bit positions; clearing a bit could break other elements
Bits are shared across elements. If "apple" and "banana" both set bit 7, clearing bit 7 when deleting "apple" would make "banana" appear absent — introducing false negatives, which Bloom filters must never have.
Q35 The optimal number of hash functions k for a Bloom filter with m bits and n expected insertions is:
A. k = (m / n) × ln 2
B. k = m / n
C. k = n / m
D. k = ln(m × n)
Correct Answer
A. k = (m / n) × ln 2
The optimal k minimizes the false positive rate. k = (m/n) × ln 2 ≈ 0.693 × (m/n). At this point, approximately half the bits in the array are set to 1.

Caching & Eviction Policies

Q36 The CLOCK algorithm approximates LRU while being cheaper to implement. How does it work?
A. It maintains a fully sorted list of items by last access timestamp, updated on every cache read
B. It evicts items in strict round-robin fashion from a ring buffer, ignoring access history
C. It maintains one reference bit per cached item
D. It counts accesses in fixed-size time windows and evicts the item with fewest accesses
Correct Answer
C. It maintains one reference bit per cached item
CLOCK arranges entries in a circular buffer with a "clock hand." Each entry has a reference bit (set to 1 on access). When evicting: if ref=1, clear to 0 and advance. If ref=0, evict it. This approximates LRU without a full sorted list.
Q37 LFU's key weakness compared to LRU for workloads with shifting popularity patterns is:
A. LFU requires significantly more memory per cached item
B. LFU cannot handle cache sizes larger than a predefined threshold
C. LFU is computationally slower on all hardware architectures
D. Past-popular items accumulate high counts and resist eviction; newly popular items get unfairly evicted
Correct Answer
D. Past-popular items resist eviction; newly popular items get unfairly evicted
LFU's Achilles heel: frequency pollution. An item accessed 10,000 times last week but never again still has count 10,000. A brand-new trending item with count 5 gets evicted first. LFU can't adapt to shifting popularity.
Q38 Increasing cache hit rate from 98% to 99% (1% absolute), given hits cost 100μs and misses cost 5,000μs, produces:
A. Approximately a 1% improvement in average latency
B. Approximately a 25% reduction in average latency (from ~198μs to ~149μs)
C. No meaningful change
D. A 50% improvement because misses are 50× more expensive
Correct Answer
B. Approximately a 25% reduction in average latency
At 98%: avg = 0.98 × 100 + 0.02 × 5000 = 198μs. At 99%: avg = 0.99 × 100 + 0.01 × 5000 = 149μs. Reduction = (198 − 149) / 198 ≈ 25%. A tiny hit rate improvement has outsized impact because misses are 50× more expensive.
Q39 The fundamental behavioral difference between FIFO and LRU is:
A. FIFO tracks access frequency; LRU tracks insertion order
B. FIFO and LRU produce identical results when all items are accessed at equal intervals
C. FIFO evicts the item inserted earliest regardless of subsequent accesses; LRU evicts the item accessed least recently, so a cache hit refreshes its position
D. LRU evicts the item inserted earliest; FIFO evicts the item accessed least recently
Correct Answer
C. FIFO evicts by insertion order; LRU evicts by last access time
The key difference: FIFO ignores cache hits — once in the queue, your position never changes. LRU promotes on every access — a hit moves the item to the front. This makes LRU better at keeping frequently-accessed items.
Q40 The LFU (Least Frequently Used) eviction policy is best suited for workloads where:
A. The set of popular items is relatively stable over time
B. New items become popular very quickly and old items rapidly lose relevance
C. Items are accessed in a strict sequential scan with no item accessed more than once
D. All items are accessed with exactly uniform probability
Correct Answer
A. The set of popular items is relatively stable over time
LFU excels when popularity is stable — the same items keep getting accessed. Frequency counts accurately reflect future demand. If popularity shifts, LFU struggles because stale high-count items block new popular ones.
Q41 Belady's Algorithm (OPT) is called the optimal eviction policy. Why is it not used in production?
A. It requires too much memory to maintain access frequency counts
B. It requires knowing the future sequence of cache accesses in advance
C. It is computationally equivalent to FIFO
D. It was designed only for read-only workloads
Correct Answer
B. It requires knowing the future sequence of cache accesses in advance
Belady's OPT evicts the item needed farthest in the future. Provably optimal but requires a crystal ball. It's used as a theoretical benchmark to evaluate how close real algorithms get to perfection.
Q42 Segmented LRU avoids cache pollution from one-hit items. How does it differ from standard LRU on first insertion?
A. New items are inserted into the middle of the LRU ordering
B. New items bypass the cache entirely on first access and are only cached after two accesses
C. New items are assigned an initial frequency count of 2
D. New items are inserted at the tail (least-recently-used end), making them first to be evicted if not re-accessed
Correct Answer
D. New items inserted at the tail (LRU end), first to be evicted if not re-accessed
Segmented LRU has two segments: probationary (new items enter at the LRU end) and protected (promoted on second access). One-hit wonders get evicted quickly from probation. Only items that prove their worth get protection.

Networking

Q43 The Physical layer (Layer 1) is primarily responsible for:
A. Assigning logical IP addresses and routing packets across multiple networks
B. Organizing bits into frames and managing local network delivery using MAC addresses
C. Converting bits into physical signals suitable for the transmission medium
D. Providing reliable end-to-end delivery using acknowledgments and retransmission
Correct Answer
C. Converting bits into physical signals suitable for the transmission medium
Layer 1 (Physical) = raw bits ↔ physical signals (electrical voltage, light pulses, radio waves). It doesn't know about addresses or reliability. A = Layer 3, B = Layer 2, D = Layer 4.
Q44 Which statement about MAC addresses is accurate according to the course material?
A. 32-bit logical addresses assigned dynamically by DHCP
B. 128-bit globally unique identifiers assigned by OS software
C. MAC addresses are globally routable
D. 48-bit hardware identifiers burned in at manufacture, used only for local network delivery
Correct Answer
D. 48-bit hardware identifiers, burned in at manufacture, local delivery only
MAC addresses are 48 bits (6 bytes, like AA:BB:CC:DD:EE:FF), set in hardware by the manufacturer, and used at Layer 2 for local delivery only. They are NOT routable across the internet — that's what IP addresses (Layer 3) are for.
Q45 QUIC (RFC 9000, used in HTTP/3) improves upon TCP in several ways. Which accurately describes QUIC's key advantages?
A. QUIC operates at the Network layer, eliminating the need for IP-level routing
B. QUIC runs over UDP, reimplements reliable delivery, integrates TLS 1.3 (0-RTT reconnection), eliminates head-of-line blocking, and survives network handoffs
C. QUIC replaces both TCP and UDP at the OS level
D. QUIC eliminates the need for encryption
Correct Answer
B. QUIC over UDP with TLS 1.3, 0-RTT, independent streams, and network handoff survival
QUIC's key innovations: (1) Built on UDP, (2) TLS 1.3 built-in with 0-RTT reconnection, (3) No head-of-line blocking — loss in one stream doesn't stall others, (4) Connection migration — switching WiFi to cellular doesn't drop the connection.
Q46 Microwave links travel faster than fiber optic cables over the same distance. The reason is:
A. Microwave signals propagate through air at close to the speed of light in a vacuum, whereas light in fiber is slowed to roughly 2/3 of vacuum speed
B. Microwave transmitters operate at higher power
C. Microwave links use fewer intermediate repeaters and switches
D. Microwave signals travel at the speed of sound in air, which exceeds photon speed in glass
Correct Answer
A. Microwave in air ≈ speed of light in vacuum; light in fiber ≈ 2/3 vacuum speed
Light in fiber travels through glass (refractive index ~1.5), slowing to about 2/3 of c. Microwave signals travel through air (refractive index ~1.0), at nearly c. HFT firms pay millions for microwave links to shave microseconds off trades.
Q47 ARP (Address Resolution Protocol) serves what purpose in the network stack?
A. Translates domain names into IP addresses
B. Dynamically assigns IP addresses to new devices
C. Bridges Layer 2 (MAC addresses) and Layer 3 (IP addresses)
D. Encrypts Ethernet frames at the Data Link layer
Correct Answer
C. Bridges Layer 2 (MAC addresses) and Layer 3 (IP addresses)
ARP answers: "I know the IP — what's the MAC?" It broadcasts on the local network: "Who has 192.168.1.5?" and the owner responds with its MAC. This bridges Layer 3 (IP) to Layer 2 (MAC). A = DNS, B = DHCP.
Q48 The Network layer (Layer 3) is responsible for:
A. Reliable, ordered byte-stream delivery between applications
B. Encoding bits as electrical or optical signals
C. Local delivery of frames using MAC addresses
D. Global routing of packets across multiple networks using IP addresses
Correct Answer
D. Global routing of packets across multiple networks using IP addresses
Layer 3 (Network) = IP addresses + routing. It moves packets across network boundaries (router to router). A = Layer 4, B = Layer 1, C = Layer 2.
Q49 TCP and UDP represent the core reliability-vs-speed tradeoff in the Transport layer. Which description is correct?
A. TCP is always lower latency than UDP
B. TCP provides reliable, ordered delivery with acknowledgments, flow control, and retransmission
C. UDP guarantees in-order delivery but leaves reliability to the application
D. TCP and UDP are interchangeable for all applications
Correct Answer
B. TCP provides reliable, ordered delivery with ACKs, flow control, and retransmission
TCP: reliable, ordered, connection-oriented (web, email, file transfer). UDP: unreliable, unordered, connectionless, but fast (video streaming, gaming, DNS). TCP guarantees delivery + ordering; UDP guarantees neither.
Q50 According to the course slides, DNS operates by default over which transport protocol and port number?
A. TCP, port 80
B. TCP, port 443
C. UDP, port 25
D. UDP, port 53
Correct Answer
D. UDP, port 53
DNS uses UDP port 53 by default because queries are small and need to be fast. Common ports to know: 53 = DNS, 80 = HTTP, 443 = HTTPS, 25 = SMTP.
