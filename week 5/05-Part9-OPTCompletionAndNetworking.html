1. OPT Worked Example (Complete)

Continuing from Part 8, here is the full OPT trace for 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5 with cache size 4. OPT looks ahead at the entire remaining sequence to decide which item to evict.

Req Result Action Cache Reasoning (future lookahead)
1 Miss Insert 1 {1} —
2 Miss Insert 2 {1, 2} —
3 Miss Insert 3 {1, 2, 3} —
4 Miss Insert 4 (full) {1, 2, 3, 4} —
1 Hit — {1, 2, 3, 4} —
2 Hit — {1, 2, 3, 4} —
5 Miss Evict 4, insert 5 {1, 2, 3, 5} Future: 1,2,3,4,5. Next use: 1→1 step, 2→2 steps, 3→3 steps, 4→4 steps (furthest)
1 Hit — {1, 2, 3, 5} —
2 Hit — {1, 2, 3, 5} —
3 Hit — {1, 2, 3, 5} —
4 Miss Evict 3, insert 4 {1, 2, 4, 5} Future: 5. Next use: 5→1 step, 1→never, 2→never, 3→never (tie; evict any)
5 Hit — {1, 2, 4, 5} —

Result: 6 misses, 6 hits out of 12 requests — this is optimal!

Hit rate: 6/12 = 50%. No other eviction policy can achieve fewer than 6 misses on this sequence with a cache of size 4.

The critical decision was at request 7 (item 5): OPT knew that item 4's next access was furthest away (4 steps), so it evicted 4. This kept items 1, 2, and 3 in cache, allowing hits on requests 8, 9, and 10. LRU also evicted 3 at this point (wrong choice), while LFU evicted 3 (also suboptimal here).

2. Final Eviction Policy Comparison

Policy Misses Hits Hit Rate
FIFO 10 2 16.7%
LRU 8 4 33.3%
LFU 7 5 41.7%
OPT 6 6 50% (optimal)
4 of the 6 misses are unavoidable cold misses (the first access to each of 1, 2, 3, 4). OPT shows that only 2 additional misses are necessary beyond the cold misses. FIFO incurs 6 extra misses, LRU incurs 4 extra, and LFU incurs 3 extra.

3. Eviction Policies in the Real World

All of the eviction policies surveyed (except OPT) are used in real systems. LRU and its approximations are the most common in practice.

Complication: Variable-Sized Items

Our model assumed all items have the same size. In reality, items can have very different sizes. A large item might cause the eviction of many small items when inserted. This makes eviction decisions much more complex — do you evict one large item or several small ones? Size-aware policies must balance the "value" of an item against the space it occupies.

The lecture references the Facebook/Meta paper "Scaling Memcache at Facebook" (NSDI 2013), which describes how Facebook uses Memcached as the foundation for a distributed key-value cache handling billions of requests per second and trillions of items for over a billion users.

4. TCP/IP 5-Layer Model — Introduction

The lecture shifts to a new major topic: networking. Understanding how data travels across the internet is essential for data systems, since databases, caches, and applications communicate over the network.

TCP/IP 5-Layer Model: The standard framework for understanding how data moves across the internet. Each layer has a specific responsibility and communicates with the layers directly above and below it.
Layer Name Responsibility Key Protocols/Concepts
5 Application User-facing protocols and data HTTP, HTTPS, SMTP, FTP, SSH, DNS
4 Transport End-to-end communication, reliability TCP, UDP, QUIC, port numbers
3 Network Routing packets across networks IPv4, IPv6, BGP, routers
2 Data Link Local network delivery, framing Ethernet frames, MAC, ARP, DHCP
1 Physical Bits as physical signals on a medium Fiber, copper, WiFi, 5G, satellite

Data flows down the stack on the sender side (encapsulation) and up the stack on the receiver side (decapsulation). Each layer wraps the data from the layer above with its own header information.

Layer 1: Physical

The physical layer converts bits to physical signals. The choice of medium determines speed, range, and latency:

  • Twisted pair (Ethernet Cat5e/6): 4 pairs, 8 conductors — spans L1 signaling + L2 framing
  • Coaxial cable (DOCSIS broadband): single copper core + shielding
  • Fiber optic (internet backbone): light pulses over km-scale range
  • Wireless spectrum: 2.4 GHz WiFi (longer range) to 5G mmWave (faster but shorter range). Higher frequency = faster but shorter range
  • Satellite: GEO has ~600ms latency (physics); LEO (e.g., Starlink) achieves 20-40ms — viable for apps
Microwave travels faster than fiber (air vs. glass). High-frequency trading firms exploit this by using microwave links between exchanges to shave microseconds off latency. Layer 1 is protocol-agnostic — a broken cable kills everything above it regardless of how good your software is.

Layer 2: Data Link

The data link layer organizes bits into frames and handles local network delivery. Key concepts:

  • Ethernet frame structure: Preamble (8B) | Dst MAC (6B) | Src MAC (6B) | Type (2B) | Payload (46-1500B) | FCS (4B)
  • MAC address: 48-bit hardware identifier burned in at manufacture (e.g., 3C:22:FB:4A:9E:01). Local only — invisible beyond the router. Switches use MAC tables; routers do not.
  • ARP (Address Resolution Protocol): bridges L2 (MAC) and L3 (IP). Broadcasts "Who has 192.168.1.1?" and gets a reply with the MAC address. ARP cache stores mappings. (Note: ARP poisoning is a known attack.)
  • DHCP: bootstraps new devices automatically with a 4-step handshake: DISCOVER → OFFER → REQUEST → ACK. Assigns IP, subnet, gateway, DNS.

Layer 3: Network

The network layer routes packets across multiple networks using IP addresses, making the global internet possible.

  • IPv4: 32-bit addresses (~4.3 billion), now exhausted. NAT extends usable addresses.
  • IPv6: 128-bit addresses (340 undecillion), eliminates NAT. Dominant on mobile.
  • IPv4 address structure: 4 octets (e.g., 192.168.1.5), split into network portion and host portion
  • BGP (Border Gateway Protocol): how autonomous systems (ISPs, companies) announce routes to each other. Trust-based — any AS can announce any prefix, which makes it vulnerable. This is exactly what caused the 2021 Facebook outage.
IP addresses are logical and hierarchical — they encode location. No single router has a complete map of the internet; each makes a local best-hop decision. BGP's trust model is its greatest vulnerability.
