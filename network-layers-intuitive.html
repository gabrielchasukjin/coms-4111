Network Layers, Intuitively

Think of the internet as a postal system. Every layer has one job.

The one idea that makes it all click: Sending data across a network is like sending a letter. You don't personally drive it to the recipient — you hand it to a system of people, each doing one job. The writer writes the letter, someone puts it in an envelope with an address, the post office routes it, the trucks carry it, and the physical roads connect the cities. Each layer only talks to the layer directly above or below it. That's the network stack.

The 5 Layers (TCP/IP Model)

We use the practical 5-layer model (what the internet actually runs on), not the theoretical 7-layer OSI model.

Layer 5 — Application
What are we saying?
You writing the letter

This is where your actual data lives — the webpage, the email body, the API response. Protocols here define the meaning of the conversation.

  • HTTP/HTTPS — "Give me this webpage"
  • SMTP — "Here's an email to deliver"
  • DNS — "What's the IP address for google.com?"
  • SSH — "Open a secure terminal session"
↓
Layer 4 — Transport
Which conversation is this for?
The envelope — labeled with sender & recipient department

Your machine runs many apps at once (browser, Spotify, email). The transport layer uses port numbers to route data to the right app. It also decides: do we care if packets arrive reliably?

  • TCP — Reliable, ordered delivery. "I'll resend anything you miss." (web, email, file transfer)
  • UDP — Fire and forget. "Speed over perfection." (video calls, gaming, DNS lookups)

Unit of data: Segment (TCP) / Datagram (UDP)

↓
Layer 3 — Network
How do we get it across the world?
The postal routing system — ZIP codes & sorting facilities

This layer handles addressing (IP addresses) and routing — finding a path from source to destination across many intermediate networks. Your data might hop through 15 routers to get from New York to Tokyo.

  • IP (Internet Protocol) — Every device gets an address (e.g., 142.250.80.46)
  • Routers live here — they read the destination IP and forward the packet one hop closer

Unit of data: Packet

↓
Layer 2 — Data Link
How do we get it to the next stop?
The local mail truck — delivers between neighboring post offices

The network layer plans the full route; the data link layer handles one hop at a time. It moves data between two directly connected devices (your laptop → your Wi-Fi router, or one router → the next router).

  • MAC addresses — hardware addresses burned into your network card (e.g., A4:83:E7:2F:01:BC)
  • Ethernet (wired) and Wi-Fi (802.11) live here
  • Switches live here — they forward frames within a local network

Unit of data: Frame

↓
Layer 1 — Physical
How do we move raw bits?
The actual roads and trucks

Electrical signals on copper wire, light pulses in fiber optic cable, radio waves for Wi-Fi. This layer doesn't know what the bits mean — it just moves 1s and 0s from point A to point B.

  • Ethernet cables, fiber optics, radio frequencies
  • Defines voltages, frequencies, connector shapes

Unit of data: Bits

Encapsulation: Russian Nesting Dolls

When you send data, each layer wraps the data from the layer above with its own header (like putting a letter in an envelope, then putting that envelope in a shipping box, then loading the box on a truck).

What actually travels on the wire:

L1 Preamble L2 Frame Header L3 IP Header L4 TCP Header Your Data L2 Trailer

Each layer only reads its own header, does its job, and passes the rest up or down.

At the destination, the process reverses: each layer strips its header and passes the contents up. The physical layer hands bits to data link, which hands a packet to network, which hands a segment to transport, which hands the data to your application. Nobody peeks at layers that aren't theirs.

The Two Questions That Separate Every Layer

Layer Addresses Scope Device
Application URLs, hostnames End-to-end meaning Your app
Transport Port numbers (e.g., :443) End-to-end reliability OS kernel
Network IP addresses Global routing (any-to-any) Router
Data Link MAC addresses Local hop (neighbor-to-neighbor) Switch
Physical None Raw signal on a wire/air Cable / antenna
The single hardest thing to internalize: IP addresses get your data across the entire internet (globally routed). MAC addresses only get your data to the next device on the same link (locally scoped). When a packet hops through 10 routers, the IP addresses stay the same the whole trip, but the MAC addresses change at every single hop.

What Happens When You Open google.com

Application: Your browser creates an HTTP GET request: "Give me the homepage"
Transport: TCP wraps it with your port (e.g., :52481) and Google's port (:443). Establishes a reliable connection.
Network: IP wraps it with your IP (192.168.1.5) and Google's IP (142.250.80.46). Routers will use this to forward it hop by hop.
Data Link: Ethernet/Wi-Fi wraps it with your MAC and your router's MAC. Just needs to reach the first hop.
Physical: Bits go out as radio waves (Wi-Fi) or electrical signals (Ethernet cable) to your router.

Remember It Like This

  • Physical = the road
  • Data Link = the local mail truck (one stop at a time)
  • Network = the ZIP code routing system (across the country)
  • Transport = the envelope label (which department/app gets it, and do we track delivery?)
  • Application = the actual letter you wrote

Data goes down the stack when sending (each layer adds a header), and up the stack when receiving (each layer strips its header). Every layer only cares about its own job.
